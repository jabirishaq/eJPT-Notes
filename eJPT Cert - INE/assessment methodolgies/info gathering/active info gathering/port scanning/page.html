Scanning the services that are on Open Ports

Enumerate the much information for the ports that are open using NMAP

-- nmap -pn <ip address>

-pn is for the systems like windows that blocks the pings and it wiont be able to do scanning.

-- nmap -pn <ip address>

this is for the lots of lots of ports and it takes time

to break it down, we need to specify the ports in the command like,

-- nmap -pn -p 80,443,3389 <ip address>

it will return the port 80,443,3389(RDP Port)

or you can add a range like

-- nmap -pn -p 1-35040 <ip address>

or can be used to scan the Fastest way

-- nmap -pn -F <ip address>

For the UDP Search:

-- nmap -pn -sU <ip address>

Can increase the verbosity

-- nmap -pn -sU <ip address> -v

For the scan for the services it

-- nmap -pn -F -sV <ip address>

For the scan of the windows version

-- nmap -pn -F -sV -O <ip address>

For the scripts and nmap scripts on the target

-- nmap -pn -F -sV -O -sC <ip address>

For Fastest of the scans we can do it

-- nmap -T<1-5> -pn -F -sV -O -sC <ip address>

keep in mind we cant use T5 it is fastest way because on production it will send the
large number of packets and web servers can lead to Denial Of Service.
